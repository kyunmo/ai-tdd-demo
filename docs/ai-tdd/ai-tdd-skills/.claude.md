## 스킬 참조 경로

에이전트는 테스트 생성 시 다음 문서들을 순서대로 참조합니다.

```
docs/ai-tdd-skills/
├── SKILL.md               ← 생성 가이드 (4단계 레벨, 판별 기준)
├── templates/             ← 계층별 테스트 템플릿
├── constraints/           ← 규칙 및 제약사항
├── references/            ← 예제 코드
└── verification/          ← 검증 절차
```

## 테스트 생성 규칙

1. 테스트 생성 시 반드시 `SKILL.md`의 4단계 레벨 비율을 적용할 것
2. `constraints/` 폴더의 모든 규칙을 준수할 것
3. `templates/`에서 클래스 유형에 맞는 템플릿을 선택할 것
4. `references/examples/`의 예제를 참고하여 일관된 스타일 유지
5. 생성 후 `verification/`의 검증 절차를 안내할 것

## 도메인 특화 규칙 (요약)

- 개인정보(주민번호, 카드번호 등) 마스킹 검증 필수
- 비밀번호는 Petra 암호화 처리 검증 필수
- 핵심 비즈니스 작업의 감사로그 기록 검증 필수
- 테스트 데이터에 실제 개인정보/자격증명 사용 금지
