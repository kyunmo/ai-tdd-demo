## 도출된 개선사항

### 4단계 테스트 레벨 우선순위 개선

**문제점**
- 기존: 테스트 수 산출 공식이 `max(10, public_methods * 3)`으로 너무 적음
- 기존: 비율(40/30/20/10) 준수로 인하여 다른 검증들을 누락하는 문제 발생
- L1 (Happy Case)이 충분하지 않음 (메서드당 1개만)

**개선 내용 (1차)**
- 테스트 수 산출 공식: `max(15, public_methods * 4)`로 상향
- L1 (Happy Case): 각 메서드 최소 2개로 필수 보장
- L2 (Edge Case): 35%로 상향
- L3 (Exception): 15%로 하향
- L4 (Mutation): 나머지 (약 10%)

**개선 내용 (2차) - L1/L4 분리**
- 테스트 수 산출 공식: `max(15, public_methods * 3)`으로 조정
- L1 (Happy Case): public_methods * 1 (정상 시나리오만)
- L2 (Edge Case): 30%
- L3 (Exception): throw_statements (throw 문 1:1)
- L4 (Mutation): public_methods * 1 (각 메서드 호출 검증 - 필수)

**개선 원칙**: L1 → L2 → L3 → L4 순서로 Coverage 확장 (비율보다 검증 충분성 우선)

### 3차 개선 - L2 vs L3 구분 기준 + L4 필수 적용

**문제점**
- L4 (Mutation) 테스트 수가 public_methods * 1 규칙을 따르지 않음
- L2 (Edge)와 L3 (Exception) 중복 테스트 발생

**개선 내용**
- L4 생성 규칙: `public_methods * 1` (각 메서드당 1개 - 필수)
- L2 vs L3 구분 기준 추가:
  - L2: 파라미터 자체의 유효성 검증 (`null`, `empty`, `0`, 음수)
  - L3: 비즈니스 로직에 의한 예외 (DB 조회 결과에 따른 예외)

### 5차 개선 - Controller 테스트 템플릿 강화

**문제점**
- 기존 Controller 테스트가 Service 테스트와 구조적으로 불일치
- Security 의존성 (JwtTokenProvider, JwtAuthenticationFilter, PasswordEncoder) 누락
- @Nested 적용이 Level 1에서 누락
- 헬퍼 메서드 및 테스트 데이터 상수 미정의
- Mutation 검증에서 argThat() 사용 미흡

**개선 내용**

1. **@AutoConfigureMockMvc(addFilters = false) 추가**
   - Security 필터 비활성화로 Controller 단위 테스트 단순화

2. **Security MockBean 3개 필수 추가**
   ```java
   @MockBean
   private com.nhcard.al.demo.security.JwtTokenProvider jwtTokenProvider;
   @MockBean
   private com.nhcard.al.demo.security.JwtAuthenticationFilter jwtAuthenticationFilter;
   @MockBean
   private org.springframework.security.crypto.password.PasswordEncoder passwordEncoder;
   ```
3. **모든 레벨에 @Nested 적용**
   - Level 1 (HappyCases)도 @Nested 클래스로 분리

4. **헬퍼 메서드 필수 정의**
   ```java
   private User createTestUser(Long id, String name, String email) { ... }
   private CreateUserRequest createTestRequest() { ... }
   ```

5. **Mutation에서 argThat() 사용한 파라미터 검증**
   ```java
   verify(userService).createUser(argThat(req ->
       req.getName().equals(TEST_NAME) &&
       req.getEmail().equals(TEST_EMAIL)
   ));
   ```
