파일을 한번에 생성하지 못하는 이유는?

● 파일을 한 번에 생성하지 못하는 이유는 Claude Code 에이전트가 파일을 만들기 전 반드시 해당 파일을 읽어 본 뒤 (또는 Read 툴을 사용해 내용을 확인한 뒤) 그 결과를 기반으로 작성하도록 설계돼 있기
때문입니다.
다음과 같은 제약이 작용합니다.

제약 사항: 읽기‑쓰기 순서                                                                         
설명: Write 또는 Edit 툴을 사용할 때는 반드시 먼저 Read 로 파일을 열어 “파일이 존재한다/내용이 어떻게 생겼는지” 를 확인해야 합니다. 읽지 않은 상태에서 바로 Write 하면 툴이 오류를 반환합니다.
────────────────────────────────────────
제약 사항: 파일 존재 여부
설명: 쓰려는 경로에 상위 디렉터리가 존재하지 않으면 Write 가 실패합니다. 테스트 파일을 만들 경우
    src/test/java/... 디렉터리가 이미 존재해야 하며, 존재하지 않으면 먼저 디렉터리를 만들(예: mkdir -p …) 필요합니다.
────────────────────────────────────────
제약 사항: 단일 파일 제한
설명: 한 번에 여러 파일을 동시에 생성하려면 각각 Write 호출을 별도로 수행해야 합니다. 현재 Write 툴은 한 파일만을 대상으로 하므로 여러 파일을 한 번에 만들 수 없습니다.
────────────────────────────────────────
제약 사항: 코드 변경 검증
설명: 에이전트는 소스 코드를 먼저 읽고 분석한 뒤 테스트 코드를 작성해야 “테스트 대상 메서드·예외·의존성”을 정확히 파악할 수 있습니다. 이를 생략하면 잘못된 테스트가 생성될 위험이 있어 자동으로 차단됩니다.
────────────────────────────────────────
제약 사항: 플랜 모드
설명: 복잡한 구현(여러 파일·디렉터리 생성·대규모 로직 변경 등)이 필요하면 EnterPlanMode 로 전환해 검토·승인을 받아야 합니다. 플랜 없이 바로 파일을 만들면 에이전트가 해당 동작을 거부합니다.    
────────────────────────────────────────
제약 사항: 툴 사용 제한
설명: Bash 명령으로 디렉터리를 만들거나 파일을 복사할 수 있지만, 파일 내용을 직접 지정하는 경우 Write 툴을 사용해야 합니다. Bash 에서 cat > … 처럼 파일을 생성하는 방식은 “파일 내용을        
전달하는 방법”이 아니므로 허용되지 않습니다.

따라서 올바른 흐름은:

1. Read 툴로 AuthService.java (또는 다른 대상 파일)을 읽는다.
2. 파일 내용과 메서드·예외 정보를 파악한다.
3. Write 툴을 사용해 src/test/java/com/nhcard/al/demo/service/AuthServiceTest.java 와 같은 테스트 파일을 하나씩 생성한다.
4. 필요 시 Write 후 TaskCreate/TaskUpdate 등으로 진행 상황을 기록한다.

이러한 절차를 따르지 않으면 “파일을 직접 생성하지 못했다”는 오류가 발생하게 됩니다. 앞으로 테스트 파일을 만들 때는 먼저 해당 소스 파일을 Read 한 뒤, Write 툴로 개별 파일을 저장하면 한 번에 정상적으로 생성할 수 있습니다.
