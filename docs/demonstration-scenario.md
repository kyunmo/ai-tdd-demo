# AI TDD 시연 시나리오: 3가지 핵심 패턴

**최종 업데이트**: 2026년 3월 23일

---

## 1. 시나리오 목표

본 시연은 AI TDD가 실제 개발 현장에서 마주하는 **3가지 핵심 과제**를 어떻게 해결하는지, 각기 다른 패턴을 통해 보여주는 것을 목표로 합니다.

| 패턴 | 과제 | AI TDD 해결책 | 비즈니스 가치 |
|---|---|---|---|
| **1. 레거시 코드 변경** | 기존 코드 변경 시 발생하는 사이드 이펙트(Side Effect)에 대한 두려움 | AI로 기존 코드에 대한 테스트 '안전망'을 빠르게 구축하여 변경에 대한 자신감 확보 | **안정성, 유지보수 비용 절감** |
| **2. 신규 기능 개발** | 처음부터 품질 높은 코드를 일관되게 작성해야 하는 부담 | AI와 함께 TDD(Test-First) 사이클을 따라가며 고품질의 코드를 설계 및 구현 | **품질, 설계 개선, 생산성** |
| **3. 외부 시스템 연동**| 타 부서/외부 시스템 개발 지연으로 인한 개발 블로킹(Blocking) | 내부 Mock API와 격리된 단위 테스트를 통해 외부 의존성을 완벽히 분리, 독립적 사전 개발 가능 | **독립성, 프로젝트 리스크 감소** |

---

## 2. 시연 준비

- 시연 전, 이전에 생성했던 `UserServiceTest.java` 파일을 삭제하여 `UserService`가 테스트 없는 레거시 코드인 상태로 준비합니다.
- `NoticeService` 또한 테스트가 없는 상태로 가정하고 시작합니다.

---

## 3막(Act) 구성의 시연 시나리오

### **1막: 레거시 코드 변경 (안전망 패턴)**

**상황**: "운영팀에서 모든 공지사항 제목 앞에 `[공지]` 라는 말머리를 붙여달라는 간단한 변경 요청이 왔습니다. 하지만 `NoticeService`는 오래된 코드라 테스트가 하나도 없어, 이 간단한 수정을 했을 때 다른 기능에 영향을 줄까 봐 불안합니다."

**1. `[Test-After]` AI로 안전망 구축**

**시연자**: "먼저, AI에게 기존 `NoticeService`의 현재 동작을 그대로 학습하고 테스트 코드를 만들어 달라고 요청하여 '안전망'부터 구축하겠습니다."

**(명령어 입력)**
```bash
> test-generator, NoticeService.java
```

**시연자**: "AI가 `NoticeServiceTest.java`를 생성했습니다. 이제 이 테스트를 실행하면, **현재 코드의 동작을 100% 보장하는 테스트들이 모두 통과(Green)**할 것입니다. 이것이 바로 우리의 안전망입니다."

**(테스트 실행하여 모두 통과(Green)하는 것을 보여줌)**

**2. 안심하고 기능 변경**

**시연자**: "이제 안전망이 생겼으니, 안심하고 요구사항에 맞게 코드를 수정하겠습니다."

**(NoticeService.java를 열어 `getNoticeById`와 `getAllNotices`의 반환 로직을 수정하여 제목 앞에 `[공지]`를 붙이도록 변경)**

```java
// 예: NoticeService.java의 getNoticeById 수정
Notice notice = noticeMapper.findById(id);
notice.setTitle("[공지] " + notice.getTitle()); // 이 라인 추가
return notice;
```

**3. 안전망으로 회귀(Regression) 감지**

**시연자**: "코드를 수정한 후, 아까 만들어 둔 테스트를 다시 실행해 보겠습니다."

**(테스트를 다시 실행. `title`을 검증하는 테스트들이 **실패(Red)**하는 것을 보여줌)**

**시연자**: "보시는 것처럼, 우리의 안전망이 **코드 변경으로 인해 기존 동작이 바뀌었다는 것을 정확하게 감지**해냈습니다. 이제 우리는 이 실패한 테스트들을 새로운 요구사항에 맞게 수정하여 다시 통과(Green) 상태로 만들면 됩니다. 이것이 레거시 코드를 안전하게 변경하는 핵심 프로세스입니다."

---

### **2막: 신규 기능 개발 (TDD 패턴)**

**상황**: "새로운 기획으로, 메인 대시보드에 **'가장 인기 있는 공지사항'**과 **'가장 활동적인 사용자'**를 보여주는 기능을 추가해야 합니다. 완전히 새로운 기능입니다."

**1. `[Red]` AI로 실패하는 테스트 먼저 생성**

**시연자**: "완전히 새로운 기능이므로, TDD(Test-Driven Development) 사이클에 따라 AI에게 **실패하는 테스트를 먼저** 만들어달라고 요청하겠습니다."

**(명령어 입력)**
```bash
> test-generator, DashboardService 테스트 코드 생성

요구사항:
- `getTopNotice()`: 조회수가 가장 높은 공지사항 1개를 반환
- `getMostActiveUser()`: 작성한 공지사항이 가장 많은 사용자 1명을 반환
```

**시연자**: "AI가 테스트 코드(`DashboardServiceTest.java`)와 비어있는 구현 클래스(`DashboardService.java`)를 생성했습니다. 당연히 구현이 없으므로, 지금 이 테스트를 실행하면 **컴파일 오류 또는 실패(Red)**가 발생합니다. 이것이 올바른 TDD의 시작입니다."

**(테스트 실행하여 실패(Red)하는 것을 보여줌)**

**2. `[Green]` 최소 기능 구현**

**시연자**: "이제 이 실패하는 테스트를 통과시킬 최소한의 코드를 `DashboardService.java`에 구현하겠습니다."

**(코드를 직접 작성하거나 미리 준비된 코드를 붙여넣기)**

**시연자**: "구현이 완료되었습니다. 다시 테스트를 실행하여 이제는 **모두 통과(Green)**하는지 확인하겠습니다."

**(테스트 실행하여 모두 통과(Green)하는 것을 보여줌)**

**3. `[Refactor]` AI로 코드 품질 리뷰**

**시연자**: "기능은 완성되었지만, TDD의 마지막은 리팩토링입니다. AI 리뷰 에이전트에게 방금 작성한 테스트 코드의 품질을 검토해달라고 요청하겠습니다."

**(명령어 입력)**
```bash
> tdd-review, DashboardServiceTest.java 리뷰
```

**시연자**: "리뷰 결과 95점으로 좋은 평가를 받았습니다. 이처럼 AI와 함께 TDD 사이클을 따르면, 처음부터 품질 높은 코드를 일관되게 작성할 수 있습니다."

---

### **3막: 외부 시스템 연동 (독립 개발 패턴)**

**상황**: "사용자 생성 시, 외부 파트너사 API를 호출하여 '외부 ID'를 받아와야 합니다. 하지만 **파트너사 API 개발이 아직 완료되지 않아** 우리 팀의 개발 일정이 지연될 위기에 처했습니다."

**1. `[Test-First]` 외부 API 연동 테스트 사전 작성**

**시연자**: "하지만 AI TDD를 활용하면 문제없습니다. 파트너사 API가 없어도, **우리가 기대하는 API 명세를 기반으로** 테스트 코드를 먼저 작성하여 우리의 로직을 사전에 완벽하게 개발하고 검증할 수 있습니다."

**(1막에서 테스트가 없도록 준비한 `UserService`를 대상으로 명령어 입력)**
```bash
> test-generator, UserService의 사용자 생성 로직에 외부 시스템 ID를 받아 저장하는 테스트 추가

요구사항:
- 사용자 생성 시 http://partners.api/v1/users API를 POST로 호출
- API 응답(JSON)에서 partnerId 값을 추출
- 새로 생성되는 User 객체의 externalId 필드에 저장
```

**2. `[Mocking]` 테스트로 외부 API 시뮬레이션**

**시연자**: "AI가 생성한 `UserServiceTest.java`를 보겠습니다. `MockRestServiceServer`라는 기술을 사용해서, **실제 네트워크 통신 없이도 외부 API 호출을 100% 시뮬레이션**하는 코드가 만들어졌습니다. 파트너사의 API가 있든 없든, 심지어 제 노트북이 인터넷에 연결되지 않아도 이 테스트는 완벽하게 동작합니다."

**(생성된 `UserServiceTest.java`의 `MockRestServiceServer` 부분을 강조하여 설명)**

**3. `[Independent Dev]` 독립적으로 기능 구현 및 검증**

**시연자**: "이제 이 테스트를 통과하도록 `UserService`에 `RestTemplate`을 사용한 API 호출 로직을 구현합니다. 파트너사의 개발 진행 상황과 완전히 독립적으로, 우리 쪽 개발을 먼저 완료하고 테스트까지 마칠 수 있습니다."

**(미리 준비된 `UserService` 구현 코드를 보여주고, 테스트를 실행하여 **모두 통과(Green)**하는 것을 보여줌)**

**시연자**: "보시는 것처럼, 우리는 파트너사 개발 지연에 전혀 영향을 받지 않고 우리에게 주어진 기능을 완벽하게 개발하고, 심지어 테스트까지 완료했습니다. 추후 실제 API가 오픈되면, 우리는 이 테스트의 URL만 실제 주소로 바꾸어 연동 테스트를 한 번만 더 수행하면 됩니다. 이것이 AI TDD가 제공하는 진정한 **'독립적 개발'**의 가치입니다."
