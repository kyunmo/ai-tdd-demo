# AI TDD 데모 프로젝트 개선 보고서

**작성일**: 2026년 3월 23일

---

## 1. 개요

본 문서는 AI TDD 데모 프로젝트에 **외부 API 연동 시뮬레이션 기능**과 **간단한 웹 UI**를 추가하는 과정에서 변경되거나 새로 생성된 모든 소스 코드의 내역을 상세히 기록한 보고서입니다. 폐쇄망 환경으로 소스 코드를 이전하거나, 변경 내역을 추적하는 데 사용될 수 있습니다.

## 2. 작업 내역 요약

| 작업 분류 | 주요 내용 | 대상 파일 |
|---|---|---|
| **의존성 추가** | `Thymeleaf` 라이브러리를 추가하여 웹 페이지 렌더링 기능 활성화 | `build.gradle` |
| **도메인 확장** | 외부 시스템 ID 저장을 위해 `User` 도메인에 `externalId` 필드 추가 | `src/main/java/nh/ai/tdd/demo/domain/User.java` |
| **Mock API 구현**| 외부 API를 흉내 내는 `MockApiController`를 프로젝트 내부에 생성 | `src/main/java/nh/ai/tdd/demo/controller/MockApiController.java` |
| **핵심 로직 수정**| `UserService`가 `RestTemplate`을 사용하여 Mock API를 호출하도록 수정 | `src/main/java/nh/ai/tdd/demo/service/UserService.java` |

## 3. 변경 및 신규 생성 파일 상세

### 3.1. 수정된 파일

#### 1. `build.gradle`
- **변경 사항**: `dependencies` 블록에 `implementation "org.springframework.boot:spring-boot-starter-thymeleaf"` 라인을 추가했습니다.
- **목적**: Thymeleaf 템플릿 엔진을 사용하여 서버 사이드에서 HTML을 렌더링하기 위함입니다.

#### 2. `src/main/java/nh/ai/tdd/demo/domain/User.java`
- **변경 사항**:
    - `private String externalId;` 필드를 추가했습니다.
    - `getExternalId()` 및 `setExternalId(String externalId)` 메서드를 추가했습니다.
- **목적**: 외부 API로부터 받은 고유 ID를 사용자 정보에 저장하기 위함입니다.

#### 3. `src/main/java/nh/ai/tdd/demo/service/UserService.java`
- **변경 사항**:
    - `RestTemplate`을 의존성으로 주입받도록 생성자를 수정했습니다.
    - `createUser` 메서드 내부에 `RestTemplate`을 사용하여 `http://localhost:8080/mock-api/external-system/user-info`를 호출하는 로직을 추가했습니다.
    - API 응답으로 받은 `externalId`를 `User` 객체에 설정하는 로직을 추가했습니다.
- **목적**: 사용자 생성 시 외부 시스템(을 흉내 내는 Mock API)과 연동하는 핵심 비즈니스 로직을 구현하기 위함입니다.

#### 4. `src/main/resources/templates/*.html` 파일들 (`index.html`, `users/list.html`, `notices/list.html`, `notices/detail.html`)
- **변경 사항**:
    - 모든 HTML 파일에서 외부 Bootstrap CDN 링크를 제거했습니다.
    - 기존 Bootstrap 클래스들을 제거하고, 기본적인 레이아웃 및 가독성을 위한 인라인 스타일 또는 순수 HTML 구조로 변경했습니다.
- **목적**: 폐쇄망 환경에서 외부 리소스에 의존하지 않고 웹 UI를 정상적으로 렌더링하기 위함입니다.

### 3.2. 신규 생성된 파일

#### 1. `src/main/java/nh/ai/tdd/demo/controller/MockApiController.java`
- **목적**: 외부 API 시스템을 시뮬레이션하는 컨트롤러입니다. 폐쇄망 환경에서 `UserService`가 호출할 대상 역할을 합니다.
- **주요 기능**: `/mock-api/external-system/user-info` GET 요청 시, 가상의 `externalId`가 포함된 JSON 응답을 반환합니다.

#### 2. `src/main/java/nh/ai/tdd/demo/config/RestTemplateConfig.java`
- **목적**: 애플리케이션 전체에서 사용할 `RestTemplate` 빈(Bean)을 생성하고 설정합니다.
- **주요 기능**: 기본적인 연결 및 읽기 타임아웃을 설정한 `RestTemplate` 인스턴스를 Spring 컨테이너에 등록합니다.

#### 3. `src/test/java/nh/ai/tdd/demo/service/UserServiceTest.java`
- **목적**: `UserService`의 단위 테스트를 수행합니다. 특히 외부 API 연동 로직을 검증하는 데 중점을 둡니다.
- **주요 기능**:
    - `MockRestServiceServer`를 사용하여 `RestTemplate`의 HTTP 호출을 가로채고, 실제 네트워크 통신 없이 가짜 응답을 반환하도록 설정합니다.
    - `createUser` 메서드 호출 시 외부 API가 정상적으로 호출되는지와, 응답받은 `externalId`가 `User` 객체에 올바르게 저장되는지 검증합니다.
    - 외부 API 호출이 실패하는 시나리오에 대한 테스트도 포함합니다.

#### 4. `src/main/java/nh/ai/tdd/demo/controller/WebController.java`
- **목적**: 사용자가 브라우저로 접근할 수 있는 HTML 페이지를 렌더링합니다.
- **주요 기능**: `/users`, `/notices`, `/notices/{id}` 등의 경로로 요청이 오면, 각 서비스에서 데이터를 조회하여 모델에 담아 해당 `Thymeleaf` 템플릿으로 전달합니다.

#### 5. `src/main/resources/templates/index.html`
- **목적**: 프로젝트의 메인 랜딩 페이지입니다.

#### 6. `src/main/resources/templates/users/list.html`
- **목적**: 전체 사용자 목록을 테이블 형태로 보여주는 페이지입니다. `externalId`도 함께 표시됩니다.

#### 7. `src/main/resources/templates/notices/list.html`
- **목적**: 전체 공지사항 목록을 리스트 형태로 보여주는 페이지입니다.

#### 8. `src/main/resources/templates/notices/detail.html`
- **목적**: 특정 공지사항의 상세 내용을 보여주는 페이지입니다.
