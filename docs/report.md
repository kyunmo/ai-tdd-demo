## 작업 이력

### 2026-02-23: docs/ai-tdd/README.md 평가 및 개선

**작업자**: AI (Claude Code)
**대상 파일**: `docs/ai-tdd/README.md`
**문서 목적**: ai-tdd 폴더의 구조와 사용법을 설명하는 **프로젝트 적용 가이드** (다른 프로젝트 작업자 대상)
**참조 문서**:
- `docs/ai-tdd/plan/2026-02-10-prd-ai-tdd.md` (PRD 최신)
- `docs/ai-tdd/plan/2026-02-09-compare-ai-tdd.md` (비교 분석)

- `docs/ai-tdd/ai-tdd-skills/GENERATION-GUIDE.md` (공통 생성 가이드)
- `docs/ai-tdd/ai-tdd-skills/.claude.md` (프로젝트 설정)

#### 문서 역할 정의

| 폴더 | 역할 | 배포 대상 |
|---|---|---|
| `ai-tdd-agent/` | Claude Code 에이전트 스킬 → `~/.claude/agents/ai-tdd-agent/`에 설치 | O |
| `ai-tdd-skills/` | 에이전트가 참조하는 규칙/템플릿/예제 → 프로젝트 `docs/ai-tdd-skills/`에 복사 | O |
| `plan/` | 기획/설계 문서 | X (내부용) |
| `README.md` | 위 폴더들의 구조와 사용법을 설명하는 적용 가이드 | O |

#### 평가 결과 (개선 전 문제점)

| # | 문제점 | 심각도 | 상세 |
|---|---|---|---|
| 1 | 두 폴더 역할 구분 부족 | 높음 | ai-tdd-agent와 ai-tdd-skills의 역할/관계/설치 위치가 명확하지 않음 |
| 2 | 디렉토리 구조 혼란 | 높음 | `nh-claude/`로 표기되어 소스 저장소인지 적용 대상 프로젝트인지 구분 불가 |
| 3 | 수동 프롬프트 예시 불필요 | 중간 | 에이전트 사용 시 프롬프트를 직접 작성할 필요 없는데, "Claude Code 프롬프트" 섹션에서 수동 프롬프트 작성법 안내 |
| 4 | 테스트 예시 코드 문제 | 중간 | `@SpringBootTest` + `@MockBean` + `@InjectMocks` 혼용 (단위테스트면 `@ExtendWith(MockitoExtension.class)` + `@Mock` 사용이 적절) |
| 5 | 시퀀스 다이어그램 단순 | 중간 | skills 참조 체인(agent→.claude.md→SKILL.md→templates→constraints)이 반영되지 않음 |
| 6 | .claude.md 가이드 부족 | 낮음 | 설정 파일 내용을 그대로 노출만 하고, 어떤 항목을 커스터마이징해야 하는지 안내 부족 |
| 7 | plan/ 폴더 미설명 | 낮음 | 배포 대상이 아닌 plan/ 폴더에 대한 설명 없음 |
| 8 | 4단계 테스트 레벨 미설명 | 낮음 | "4단계 테스트 접근 방식" 언급만 있고 구체적인 레벨별 설명 없음 |

#### 변경 요약

- **문서 구조**: 비체계적 나열 → 7개 섹션(개요/설치/사용법/예제/참조체계/기능/주의사항)으로 체계화
- **관점 교정**: "적용 가이드" 목적에 맞게 설치→설정→사용→검증 순서로 재구성
- **시각화**: Mermaid 다이어그램 2개 (동작 흐름 시퀀스, 스킬 참조 체계)
- **예시 코드**: 단위테스트 어노테이션 교정, 4단계 레벨 구분 주석 추가
- **불필요 섹션 제거**: 수동 프롬프트 작성 예시 삭제 (에이전트가 자동 처리)
