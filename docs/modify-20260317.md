### 5-7. 테스트 수 산출 공식 자기참조 문제 해결

#### `.claude/agents/test-generator.md`

기존 공식이 "L4 = public 메서드 수" 를 정의하면서 전체 수를 "L1+L2+L3+L4"로 계산하는데,
L2/L3의 산출 근거가 "소스 코드 분석 결과에 따라"로만 기술되어 자기참조 구조였음.

수정 후:
```
L1 = public 메서드 수 (각 메서드당 Happy Case 1개)
L2 = 분기(if/switch) 수 + 파라미터 경계값 수 (Edge Case)
L3 = throw 문 수 + catch 블록 수 (Exception Case)
L4 = public 메서드 수 (각 메서드당 Mutation 검증 1개)

최소 테스트 수 = L1 + L2 + L3 + L4
```

각 레벨의 산출 근거를 소스 코드의 정량적 지표(public 메서드 수, 분기 수, throw 수)에 직접 연결하여 자기참조 제거.
