## 4. test-generator.md와 generation-guide.md 중복 (Medium)

두 문서의 내용이 **70% 이상 중복**됨.

| 중복 영역 | test-generator.md | generation-guide.md |
|---|---|---|
| 클래스 유형 판별 | 3.2절 | 1.1절 |
| 4단계 레벨 정의 | 3.6절 | 2절 |
| 테스트 수 산출 공식 | 3.6절 | 없음 (agent만) |
| L2 vs L3 구분 | 3.6절 | 2절 |
| 반환타입별 패턴 | 3.6절 | 3.2절 |
| 파라미터별 Edge Case | 3.6절 | 3.1절 |
| 생성 결과물 구조 | 3.6절 | 4절 |

**제안**: `generation-guide.md`가 "규칙/기준의 원본"이고, `test-generator.md`는 이를 참조하는 형태로 역할 분리. 현재는 양쪽 모두 상세 내용을 가지고 있어 수정 시 동기화 비용 발생.

## 5. tdd-review.md 문제 (Medium)

- **전체 영문** — 프로젝트의 다른 문서 모두 한글인데 이것만 영문
- **Generic 내용** — NH 도메인 규칙, 4단계 레벨 체계와 **전혀 연결 없음**
- **ai-tdd-skills 미참조** — constraints, templates 등을 참조하지 않음
- CLAUDE.md의 에이전트 테이블에 "예정" 상태로 등록되어 있으나 이미 파일이 존재

## 7. 구조적 개선 제안

| # | 제안 | 이유 | 우선순위 |
|---|---|---|---|
| 1 | **누락 예제 2개 추가** (`mapper-test-example.md`, `util-test-example.md`) | agent가 참조하도록 되어 있으나 파일 없음 | High |
| 2 | **test-generator.md ↔ generation-guide.md 역할 분리** | 중복 제거, 단일 진실 원천(SSOT) 확보 | High |
| 3 | **controller-test.md의 Security MockBean 하드코딩 제거** | `com.nhcard.al.demo.security.*` FQCN이 하드코딩 → 다른 프로젝트 적용 시 깨짐 | Medium |
| 4 | **tdd-review.md 한글화 + ai-tdd-skills 연동** | 현재 프로젝트 체계와 분리되어 있음 | Medium |
| 5 | **test-coverage.md의 Gradle 설정 예시를 6.8.3 호환으로 변경** | `xml.required` → `xml.enabled` | Medium |
| 6 | **L1의 `@Nested` 사용 일관성 통일** | `generation-guide.md`/`test-generator.md`에서는 L1이 `@Nested` 없이, `service-test.md`/예제에서는 `@Nested` 사용 → 통일 필요 | Low |
| 7 | **테스트 수 산출 공식 검증** | `총 = max(15, methods*3)` 인데 L2 = `ceil(총*0.30)` → 자기참조적. L1+L3+L4의 합 이후 나머지를 L2로 배분하는 게 명확 | Low |
