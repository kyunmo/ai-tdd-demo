  최우선 개선 순서

  1. 누락된 Edge Case(null password, 음수 ID 등) 보강 – 테스트 실패를 사전에 차단합니다.
  2. Exception 메시지 검증 추가 – 오류 원인 파악이 쉬워지고, 비즈니스 로직과 오류 처리 일관성을 확인합니다.
  3. Mutation 테스트 확대 – updateUser, deleteUser 와 같은 메서드도 부수 효과가 최소임을 검증합니다.
  4. 캡처 검증(ArgumentCaptor) 도입 – 서비스가 실제 전달하는 객체 필드값을 확인해 데이터 무결성을 보장합니다.

  위 권고 사항을 반영하면 테스트 품질이 향상되고, TDD 원칙(실패→코드 작성→테스트 통과) 에 더욱 부합하게 됩니다.

  ※ 현재 테스트는 모두 통과 기준을 만족하고 있어 빌드 단계에서 실패가 발생하지는 않습니다. 다만, 위와 같은 추가 검증을 통해 리팩터링이나 새로운 요구사항이 들어왔을 때 회귀 위험을 최소화할 수 있습니다.
