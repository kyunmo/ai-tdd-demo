## 1. SKILL.md 점검 결과

### 개선 전 문제점

| # | 문제점 | 심각도 |
|---|---|---|
| 1 | **에이전트 동작 내용 혼재**: "에이전트 실행 지침"(4장), "에이전트 성공 지표"(5장)가 포함됨. 이 내용은 `ai-tdd-agent/SKILL.md`의 역할이지 참조용 스킬 가이드의 역할이 아님 | 높음 |
| 2 | **클래스 유형 판별 기준 부재**: "서비스 → service-test.md"라는 매핑만 있고, 어노테이션(`@Service`, `@RestController` 등) 기반 판별 기준이 없음 | 높음 |
| 3 | **4단계 레벨이 추상적**: 레벨별 구체적인 생성 기준/패턴/예시가 없어 에이전트가 참조하기 어려움 | 중간 |
| 4 | **규칙 적용 우선순위 미명시**: constraints/ 파일 4개를 나열만 하고 적용 순서/충돌 시 우선순위가 없음 | 중간 |
| 5 | **생성 결과물 구조 미정의**: 생성되는 테스트 클래스의 표준 구조(어노테이션, 섹션 구분 등)가 없음 | 중간 |
| 6 | **비현실적 지표**: "테스트 생성 30초", "사용자 만족도 4.5점" 등 AI 참조 문서에 부적합한 내용 | 낮음 |
| 7 | **검증 기준 불명확**: verification/ 참조만 하고 합격/불합격 기준이 없음 | 낮음 |

### 개선 내용

| # | 개선 항목 | 반영 섹션 |
|---|---|---|
| 1 | 에이전트 동작 내용(4장, 5장) 제거 → 순수 참조 가이드로 역할 정립 | 전체 |
| 2 | 어노테이션 기반 클래스 판별 기준 + 판별 우선순위(어노테이션 > 클래스명 > 패키지) 추가 | 1.1 |
| 3 | 4단계 레벨별 **목적/대상/패턴/생성기준/예시패턴** 상세화 | 2장 |
| 4 | 규칙 적용 순서 테이블 + 충돌 시 우선순위 명시 ("nh-rules가 최우선") | 1.2 |
| 5 | 생성 결과물 표준 구조 (코드 스켈레톤, 유형별 어노테이션, 필수 검증 항목) 추가 | 3장 |
| 6 | NH 도메인 특화 검증 항목 별도 섹션 추가 | 4장 |
| 7 | 검증 프로세스에 합격/불합격 기준 및 불합격 시 조치 추가 | 5장 |

## 2. document-guide.md 점검 결과

### 개선 전 문제점

| # | 문제점 | 심각도 |
|---|---|---|
| 1 | **각 문서 설명이 피상적**: 모든 문서가 "목적/내용/사용방법" 3줄로만 구성. 핵심 내용이 뭔지 알 수 없음 | 높음 |
| 2 | **문서 간 참조 관계 미설명**: 에이전트가 어떤 순서로 어떤 문서를 참조하는지 흐름이 없음 | 높음 |
| 3 | **공통화/커스터마이징 구분 없음**: 프로젝트 적용 시 어떤 파일을 수정하고 어떤 파일은 그대로 쓰는지 안내 없음 | 중간 |
| 4 | **Mermaid 다이어그램이 단순**: 참조 흐름이 아닌 단순 나열 구조 | 낮음 |
| 5 | **문서 업데이트 가이드 없음**: 새 프로젝트 적용, 규칙 추가 등 상황별 대응 안내 없음 | 낮음 |

### 개선 내용

| # | 개선 항목 | 반영 섹션 |
|---|---|---|
| 1 | 각 문서별 **역할/핵심내용/커스터마이징 여부**를 테이블로 정리, templates/는 판별기준+테스트방식까지 포함 | 2장 |
| 2 | 에이전트 참조 흐름 Mermaid 다이어그램 추가 (.claude.md → SKILL.md → 판별 → templates → constraints → references → 생성 → verification) | 1.2 |
| 3 | 공통화 수준 요약 테이블 추가 (100% 공통 / 90% 공통 / 템플릿별 구분) | 3장 |
| 4 | constraints/ 적용 우선순위 순서 반영 | 2.4 |
| 5 | 문서 업데이트 가이드 (상황별 대상 문서/작업 + 주의사항) 추가 | 4장 |

| 3 | 프로젝트 구조 테이블 추가 (기본패키지, 소스/테스트/리소스 경로) | 프로젝트 구조 |
| 4 | 버전 정보 추가 (프레임워크 버전, JDK, 빌드도구 버전) | 프로젝트 정보 |
| 5 | 스킬 참조 경로 + 테스트 생성 규칙 추가 (에이전트에게 다음 단계 안내) | 스킬 참조 경로, 테스트 생성 규칙 |

## 4. 변경 요약

### 파일별 변경 규모

| 파일 | 개선 전 | 개선 후 | 변경 수준 |
|---|---|---|---|
| `SKILL.md` | 5개 섹션 (에이전트 내용 혼재) | 5개 섹션 (순수 생성 가이드) | **전면 재작성** |
| `document-guide.md` | 6개 섹션 (피상적 설명) | 4개 섹션 (상세 + 관계 + 커스터마이징) | **전면 재작성** |
| `.claude.md` | 영문 4개 섹션 | 한글 7개 섹션 | **전면 재작성** |

### 핵심 개선 포인트

1. **역할 정립**: SKILL.md에서 에이전트 동작 내용을 분리하여 순수 참조 가이드로 정립
2. **구체성 확보**: 4단계 레벨별 생성 기준/패턴/예시를 에이전트가 실제 참조 가능한 수준으로 상세화
3. **판별 체계화**: 어노테이션 기반 클래스 판별 + 우선순위(어노테이션 > 클래스명 > 패키지)
4. **커스터마이징 안내**: .claude.md에 `[수정필요]` 태그, document-guide에 공통화 수준 테이블
5. **참조 흐름 시각화**: 에이전트의 문서 참조 순서를 Mermaid로 명확화
6. **언어 일관성**: .claude.md를 영문 → 한글로 전환

## 5. templates/ 폴더 점검 결과

### 공통 문제점 (4개 파일 동일)

| # | 문제점 | 심각도 |
|---|---|---|
| 1 | **오타**: "헤ppy" - 한/영 혼재 오타가 4개 파일 모두 동일 | 높음 |
| 2 | **4개 파일이 거의 동일**: 역할/요구사항/출력형식/제약사항이 copy-paste 수준, 계층별 차별화 없음 | 높음 |
| 3 | **코드 스켈레톤 없음**: 어노테이션, Mock 설정, 테스트 패턴 등 구체적 구조가 없어 에이전트 참조 불가 | 높음 |
| 4 | **계층별 특화 패턴 부재**: Controller의 MockMvc, Mapper의 DB 테스트, Util의 ParameterizedTest 등 미반영 | 중간 |
| 5 | **4단계 레벨 계층별 예시 없음**: 각 계층에 맞는 Happy/Edge/Exception/Mutation 구체적 예시 부재 | 중간 |
| 6 | **제약사항이 일반적**: 4개 파일 모두 동일한 4줄 제약사항. 계층별 특화 제약이 없음 | 낮음 |

### 6.1. nh-rules.md

#### 개선 전 문제점

| # | 문제점 | 심각도 |

## 9. ai-tdd-agent/SKILL.md 점검 결과 (가장 중요한 파일)

### 개선 전 문제점

| # | 문제점 | 심각도 |

| 2 | **행동 지시 부재**: "이 요청을 받으면 → 이렇게 행동하라"는 실제 지시가 전혀 없음 | **치명** |

### 개선 핵심: "기능 소개" → "행동 지시서"로 전환

**개선 전**: 에이전트에게 "이 스킬은 이런 기능이 있습니다"라고 설명 → 에이전트가 무엇을 해야 하는지 모름

**개선 후**: 에이전트에게 "이 요청을 받으면, 이 순서로, 이 문서를 읽고, 이렇게 행동하라"라고 지시 → 에이전트가 정확히 실행

| # | 개선 항목 | 반영 섹션 |

### 공통 개선 방향

3. **실행 가능성**: "확인하세요" → 구체적 명령어 + 합격/불합격 기준 + 조치 순서
4. **일관성**: 영문/한글 혼재 → 전체 한글, Given-When-Then 통일, 오타 제거
5. **역할 분리**: 문서 간 내용 중복 → 명확한 역할 구분 + 참조 유도
6. **레퍼런스 완성**: 4개 계층 모두 완전한 예제 제공 (소스 + 테스트 + 해설)
