## 7. 측정 방법 가이드

실제 수치를 채우기 위한 데이터 수집 방법입니다.

### 7.1. 커버리지 수치

```bash
# JaCoCo 리포트 생성
./gradlew test jacocoTestReport

# 리포트 위치
build/reports/jacoco/test/html/index.html

# 전체 수치: index.html의 Total 행
# 레이어별: 패키지별 행
```

### 7.2. 테스트 코드 규모

```bash
# 테스트 클래스 수
find src/test -name "*Test.java" | wc -l

# 테스트 메서드 수
grep -r "@Test" src/test --include="*.java" | wc -l

# 테스트 코드 라인 수
find src/test -name "*.java" -exec cat {} + | wc -l

# 소스 코드 라인 수
find src/main -name "*.java" -exec cat {} + | wc -l
```

### 7.3. 레벨별 분포

```bash
# 각 레벨 테스트 수 (주석 기반)
grep -r "Level 1" src/test --include="*.java" | wc -l
grep -r "Level 2" src/test --include="*.java" | wc -l
grep -r "Level 3" src/test --include="*.java" | wc -l
grep -r "Level 4" src/test --include="*.java" | wc -l
```

### 7.4. 자율 실행 성공률

AI TDD 에이전트 실행 시 각 단계의 성공/실패를 기록합니다.
배치 실행 종합 리포트(`batch-execution.md`)의 결과를 집계하여 산출합니다.
