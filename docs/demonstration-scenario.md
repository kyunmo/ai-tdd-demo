# AI TDD 시연 시나리오: 3가지 핵심 패턴 (가독성 개선안)

**최종 업데이트**: 2026년 3월 23일

---

## 1. 시나리오 목표

본 시연은 AI TDD가 실제 개발 현장에서 마주하는 **3가지 핵심 과제**를 어떻게 해결하는지, AI와 개발자의 협업 관점에서 명확하게 보여주는 것을 목표로 합니다.

| 막(Act) | 핵심 패턴 | 시연 목표 |
|:---:|---|---|
| **1막** | **레거시 코드 변경** | AI로 '안전망'을 구축하여 기존 코드를 자신감 있게 변경하고, 의도치 않은 변경을 즉시 감지하는 과정 시연 |
| **2막** | **신규 기능 개발** | AI가 제안한 '초안'을 기반으로 개발자가 '핵심 로직'을 주입하여 TDD 사이클을 가속화하는 과정 시연 |
| **3.막**| **외부 시스템 연동**| 외부 의존성을 완벽히 분리하여, 파트너사를 기다리지 않고 독립적으로 사전 개발 및 실시간 검증하는 과정 시연|

---

## 2. 시연 준비

- **1막 준비**: `NoticeServiceTest.java` 파일이 존재하지 않는지 확인합니다. (있으면 삭제)
- **3막 준비**: `users/list.html`에 "외부 ID 일괄 조회" 버튼을 추가하고, 이와 연결될 `UserService.populateExternalIds()` 및 컨트롤러 엔드포인트를 미리 구현해 둡니다.

---

## **1막: 레거시 코드 변경 (안전망 패턴: 검토 및 강화)**

> **상황**: "운영팀에서 모든 공지사항 제목 앞에 `[공지]` 라는 말머리를 붙여달라는 간단한 변경 요청이 왔습니다. 하지만 `NoticeService`는 테스트가 하나도 없어 불안합니다."

---

#### **1단계: AI로 테스트 뼈대 생성**

🎙️ **(멘트)**: "AI TDD의 첫 번째 패턴, 레거시 코드 변경입니다. 먼저 AI에게 아무런 추가 조건 없이, `NoticeService`의 테스트를 만들어 달라고 간단하게 요청하겠습니다."

💻 **(행동)**:
1.  Claude Code에 프롬프트를 입력합니다.
    ```bash
    > test-generator, NoticeService.java
    ```

---

#### **2단계: 개발자의 통찰력으로 안전망 강화**

🎙️ **(멘트)**: "AI가 테스트의 기본 구조를 빠르게 만들어주었습니다. 하지만 **여기서 개발자의 역할이 중요합니다.** 우리 비즈니스에서 공지사항의 핵심은 **'제목'**입니다. AI가 만든 뼈대 위에, 우리의 비즈니스 지식을 담아 이 **안전망을 더 튼튼하게 강화**해 보겠습니다."

💻 **(행동)**:
1.  생성된 `NoticeServiceTest.java`를 엽니다.
2.  `getNoticeById` 테스트의 `Then` 부분에 제목을 검증하는 `assertThat` 한 줄을 실시간으로 추가합니다.
    ```java
    assertThat(result.getTitle()).isEqualTo("원본 제목"); // <-- "이것이 핵심"이라고 강조
    ```

---

#### **3단계: 강화된 안전망으로 기준점 설정 (Green)**

🎙️ **(멘트)**: "이제 우리의 비즈니스 지식까지 반영된 **완벽한 안전망이 완성**되었습니다. 이 테스트를 실행해서 현재 상태가 정상(Green)임을 확인하고, 이 상태를 우리의 '기준점'으로 삼겠습니다."

💻 **(행동)**:
1.  수정된 테스트를 실행하여 모두 **통과(Green)**하는 것을 보여줍니다.

---

#### **4단계: 기능 변경 및 회귀 감지 (Red)**

🎙️ **(멘트)**: "이제 이 튼튼한 안전망을 믿고, 제목에 `[공지]`를 붙이는 코드를 수정하겠습니다. 수정 후에 안전망을 다시 실행해 보겠습니다."

💻 **(행동)**:
1.  `NoticeService.java`를 수정하여 제목을 변경하는 로직을 추가합니다.
2.  테스트를 다시 실행하여, 방금 추가했던 바로 그 라인에서 **실패(Red)**하는 것을 명확하게 보여줍니다.

🎙️ **(멘트)**: "**보십시오. AI가 만든 뼈대에 우리가 추가한 단 하나의 핵심 검증 로직이, 이 중요한 변경 사항을 정확하게 잡아냈습니다.** AI는 90%의 반복작업을, 개발자는 10%의 핵심 통찰력을 더해 완벽한 안전망을 만드는 것. 이것이 AI 시대의 현명한 개발 방식입니다."

---
---

## **2막: 신규 기능 개발 (TDD 패턴: AI 초안 기반 개발)**

> **상황**: "새로운 기획으로, 메인 대시보드에 **'가장 인기 있는 공지사항'**과 **'가장 활동적인 사용자'**를 보여주는 기능을 추가해야 합니다."

---

#### **1단계: AI로 테스트와 '구현 초안' 생성 (Red)**

🎙️ **(멘트)**: "두 번째 패턴, 신규 기능 개발입니다. AI에게 요구사항을 전달하여 **실패하는 테스트를 먼저** 만들어 달라고 요청하겠습니다."

💻 **(행동)**:
1.  Claude Code에 요구사항과 함께 프롬프트를 입력합니다.
    ```bash
    > test-generator, DashboardService 테스트 코드 생성

    요구사항:
    - `getTopNotice()`: 조회수가 가장 높은 공지사항 1개를 반환
    - `getMostActiveUser()`: 작성한 공지사항이 가장 많은 사용자 1명을 반환
    ```
2.  생성된 `DashboardServiceTest.java`와 `DashboardService.java` 초안을 잠시 보여줍니다.

🎙️ **(멘트)**: "AI가 테스트뿐 아니라 구현 코드의 '초안'까지 생성했습니다. 하지만 이 초안은 '조회수가 가장 높다'는 핵심 요구사항을 모르므로, 테스트는 당연히 **실패(Red)**합니다."

💻 **(행동)**:
1.  테스트를 실행하여 **실패(Red)**하는 것을 보여줍니다.

---

#### **2단계: AI 초안에 '핵심 비즈니스 로직' 주입 (Green)**

🎙️ **(멘트)**: "바로 이 지점에서 개발자의 역할이 빛을 발합니다. AI가 마련해준 구조 위에, **'가장 높은 조회수'라는 핵심 비즈니스 로직을 주입하는 것**은 개발자의 몫입니다."

💻 **(행동)**:
1.  `DashboardService.java`의 내용을 미리 준비된 코드로 수정합니다.
2.  테스트를 다시 실행하여 모두 **통과(Green)**하는 것을 보여줍니다.

🎙️ **(멘트)**: "**AI는 반복적인 구조를, 개발자는 창의적인 비즈니스 로직을 담당하는 것.** 이것이 AI와 협업하여 신규 기능을 빠르게 개발하는 방법입니다."

---

## **3막: 외부 시스템 연동 (독립 개발 패턴: 실시간 증명)**

> **상황**: 사용자 정보에 '외부 시스템 ID' 필드가 비어있습니다. 파트너사 API를 호출해서 이 필드를 채워야 하는데, 파트너사 개발이 아직 완료되지 않아 프로젝트가 지연될 위기입니다.

---

#### **1단계: 현재 상태 확인 (Before)**

🎙️ **(멘트)**: "마지막 세 번째 패턴, 외부 시스템 연동입니다. 파트너사 API가 개발 지연 중인 상황을 가정해 보겠습니다. 먼저 현재 사용자 목록 페이지를 보시죠."

💻 **(행동)**:
1.  브라우저에서 `/users` 페이지를 엽니다.
2.  '외부 시스템 ID' 컬럼이 비어있는 것을 청중에게 명확히 보여줍니다.

🎙️ **(멘트)**: "**보시는 것처럼 ID가 비어있습니다.** 지금부터 이 기능을, 파트너사를 기다리지 않고 TDD로 먼저 개발해 보겠습니다."

---

#### **2단계: AI와 함께 독립적인 TDD 개발**

🎙️ **(멘트)**: "이 비어있는 ID를 채우는 `populateExternalIds` 라는 새로운 기능을 TDD로 개발합니다. 시간 관계상 전체 TDD 사이클 대신, AI와 함께 했을 때의 핵심 결과물을 보여드리겠습니다."

💻 **(행동)**:
1.  미리 준비된 `UserServiceTest.java` 파일을 열어 `MockRestServiceServer`를 사용한 테스트 코드를 강조합니다.

🎙️ **(멘트)**: "**AI가 API 명세만으로 이처럼 완벽한 테스트 환경을 구축**해 주었습니다. 실제 네트워크 통신 없이도, 마치 파트너사 API가 존재하는 것처럼 테스트할 수 있습니다."

💻 **(행동)**:
1.  미리 준비된 `UserService.java`의 `populateExternalIds` 메서드 구현을 보여줍니다.
2.  해당 테스트를 실행하여 **성공(Green)**하는 것을 보여줍니다.

🎙️ **(멘트)**: "이 테스트 환경 덕분에, 우리는 파트너사의 개발 진행 상황과 **100% 독립적으로 우리 기능을 먼저 개발하고, 심지어 검증까지 완벽하게 마쳤습니다.**"

---

#### **3단계: 실시간 결과 증명 (After)**

🎙️ **(멘트)**: "자, 이제 이 기능이 정말 동작하는지, UI를 통해 **직접, 그리고 실시간으로 증명해 보겠습니다.**"

💻 **(행동)**:
1.  다시 브라우저의 `/users` 목록 페이지로 돌아갑니다.
2.  "외부 ID 일괄 조회" 버튼을 가리킵니다.

🎙️ **(멘트)**: "제가 이 버튼을 클릭하면, 방금 TDD로 개발한 백엔드 로직이 실행될 것입니다. 내부적으로는 저희가 만든 Mock API 서버를 호출해서, 비어있는 사용자들의 외부 ID를 가져와 DB에 업데이트하게 됩니다."

💻 **(행동)**:
1.  **"외부 ID 일괄 조회" 버튼을 클릭합니다.**
2.  페이지가 새로고침되면서, 이전에 비어있던 '외부 시스템 ID' 필드가 `ext-...` 값으로 **실시간으로 채워지는 것을 극적으로 보여줍니다.**

🎙️ **(멘트)**: "**보십시오! 파트너사는 아직 코드를 한 줄도 작성하지 않았지만, 우리는 이미 외부 연동 기능 개발을 완료하고, 눈 앞에서 그 결과를 증명했습니다.** 이것이 바로 AI TDD가 제공하는 진정한 **'독립적 개발'**과 **'사전 검증'**의 가치입니다."
