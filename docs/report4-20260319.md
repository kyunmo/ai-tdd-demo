# AI-TDD 스킬 세트 개선 제안 보고서

**문서 번호**: report4-20260319  
**작성자**: AI Agent  
**작성일**: 2026년 3월 19일  

---

## 1. 개요

본 문서는 `docs/ai-tdd-skills/`에 정의된 AI-TDD 스킬 세트의 현황을 분석하고, 핵심 목표인 **일관된 테스트 코드 생성**, **자동화된 생성-빌드-검증 사이클 강화**, 그리고 **생성 결과물의 품질 향상**을 위한 구체적인 개선 방안을 제안합니다.

현재 스킬 세트는 매우 상세하고 체계적으로 구성되어 있으나, 일부 영역에서 AI 모델의 해석 차이를 줄이고, 자동화 흐름을 더욱 견고하게 만들며, 테스트 품질을 한 단계 높일 수 있는 기회가 존재합니다.

---

## 2. 핵심 목표별 현황 분석 및 개선 제안

### 2.1. 일관된 테스트 코드 생성 (Consistency)

**현황 분석**:
- `generation-guide.md`와 각 계층별 템플릿은 매우 상세한 규칙을 제공하여 일관성 확보에 크게 기여하고 있습니다.
- `naming-conventions.md`와 `code-style.md`는 코드의 구조적 일관성을 잘 정의하고 있습니다.
- 최근 `test-generator.md`에 추가된 "문서 참조 및 규칙 적용 원칙"과 "테스트 수 산출 공식" 명확화는 AI 모델의 해석 차이를 줄이는 데 긍정적인 영향을 줄 것입니다.
- 하지만, 여러 문서에 분산된 규칙들을 AI가 종합적으로 해석하고 우선순위를 적용하는 과정에서 여전히 일부 모델(e.g., gpt-oss-120b)이 "불필요한 단계"를 생성하는 등의 비일관성이 발생하고 있습니다.

**개선 제안**:

1.  **핵심 규칙 통합 및 요약 문서 신설**:
    - **제안**: `constraints/` 폴더에 `priority-rules-summary.md` 파일을 신설합니다.
    - **내용**: 여러 문서에 흩어져 있는 최우선 규칙(NH 도메인 규칙, 4단계 레벨, 테스트 수 공식, 예외 처리 패턴 등)의 핵심만 요약하여 제공합니다. `test-generator.md`는 AI에게 이 요약 문서를 다른 상세 문서보다 먼저 읽고 메모리에 상주시키도록 지시합니다.
    - **기대 효과**: AI가 가장 중요한 제약 조건을 명확히 인지하게 하여, 모델 종류에 상관없이 일관된 결과물을 생성하도록 유도하고 "불필요한 단계" 생성을 억제합니다.

2.  **L2(Edge Case) 테스트 생성 기준 구체화**:
    - **제안**: `generation-guide.md`의 "파라미터 타입별 Edge Case 상세 매트릭스"를 더 구체화합니다.
    - **내용**: `String` 타입의 경우, `null`, `""`, `"   "`(공백), `최대길이 초과 문자열`, `특수문자 포함 문자열` 등 테스트해야 할 케이스를 명시적으로 나열합니다.
    - **기대 효과**: "나머지" 또는 "충분성"과 같은 추상적인 기준 대신 명확한 체크리스트를 제공하여, AI가 일관된 수와 종류의 Edge Case 테스트를 생성하도록 합니다.

### 2.2. 자동화 사이클 강화 (Automation Cycle)

**현황 분석**:
- `verification/` 폴더 내 문서들은 컴파일, 실행, 커버리지 검증을 위한 명확한 Gradle 명령어와 기준을 제시하고 있어 자동화의 기반이 잘 마련되어 있습니다.
- `test-generator.md`의 "검증 및 수정" 섹션은 실패 시 자가 수정을 시도하는 지능적인 자동화 사이클을 정의하고 있습니다.

**개선 제안**:

1.  **통합 실행 스크립트 도입**:
    - **제안**: `verification/` 폴더에 `run-ai-tdd-cycle.sh` 셸 스크립트 템플릿과 가이드를 추가합니다.
    - **내용**: 이 스크립트는 `클래스명`을 인자로 받아 `테스트 생성 → 컴파일 → 실행 → 커버리지 검증` 전 과정을 순차적으로 실행하고, 각 단계의 성공/실패 결과를 명확하게 출력합니다.
    - **기대 효과**: AI가 여러 명령어를 순차적으로 실행하는 대신 단일 스크립트를 호출하도록 하여, 전체 사이클 실행의 안정성과 신뢰도를 높입니다. 실패 지점을 명확히 파악하여 후속 조치(수정 또는 보고)를 더 용이하게 합니다.

2.  **`test-generator.md`의 실패 처리 로직 강화**:
    - **제안**: `test-generator.md`의 "재시도 정책"을 더 구체화합니다.
    - **내용**: "3회 실패 시 테스트 재생성" 규칙에 더해, "컴파일 실패 시, 실패 로그를 바탕으로 `verification/compile-check.md`의 해결 가이드를 참조하여 수정 시도" 와 같은 구체적인 디버깅 및 자가 수정 로직을 추가합니다.
    - **기대 효과**: 막연한 재시도 대신, 문서화된 해결책을 기반으로 AI가 더 지능적으로 실패에 대응하여 자동화 사이클의 완결성을 높입니다.

### 2.3. 생성 결과물 품질 향상 (Quality)

**현황 분석**:
- 4단계 테스트 레벨은 테스트의 다양성과 깊이를 보장하는 훌륭한 품질 프레임워크입니다.
- NH 도메인 규칙은 비즈니스 로직의 특수성을 반영하여 테스트 품질을 높입니다.
- `references/examples/`의 예제는 좋은 품질의 테스트 코드 스타일을 제시합니다.

**개선 제안**:

1.  **비동기 및 시간 의존적 로직 테스트 가이드 추가**:
    - **제안**: `constraints/advanced-patterns.md` 파일을 신설하고, `references/examples/`에 관련 예제를 추가합니다.
    - **내용**: `@Async`와 같은 비동기 메서드 테스트 방법(e.g., `Awaitility` 사용), `LocalDateTime.now()`와 같이 시간에 의존적인 코드를 테스트하는 방법(e.g., `Clock` 객체 주입 및 Mocking)에 대한 가이드와 예제를 제공합니다.
    - **기대 효과**: 현재 스킬 세트가 다루지 않는 복잡한 시나리오에 대한 테스트 생성 능력을 부여하여, 테스트 커버리지의 질을 한 단계 높입니다.

2.  **L4(Mutation) 테스트 강화**:
    - **제안**: `generation-guide.md`에서 L4 테스트의 필수성을 강조하고, 더 다양한 `verify` 패턴을 예시로 추가합니다.
    - **내용**: `ArgumentCaptor`를 사용하여 메서드에 전달된 객체의 내부 필드 값을 정밀 검증하는 패턴, `InOrder`를 사용하여 호출 순서를 검증하는 패턴을 L4의 핵심 검증 항목으로 명시합니다.
    - **기대 효과**: 단순 호출 여부를 넘어, 로직의 내부 동작과 흐름을 더 정밀하게 검증하는 고품질 테스트 생성을 유도합니다.

---

## 3. 결론 및 기대효과

제안된 개선 사항들은 기존의 잘 구성된 AI-TDD 스킬 세트를 한 단계 발전시키는 데 중점을 둡니다. **규칙의 명확화와 통합**을 통해 AI 모델 간의 성능 편차를 줄여 **일관성**을 높이고, **스크립트 기반의 통합 실행**으로 **자동화 사이클**을 더욱 견고하게 만들며, **고급 테스트 패턴 도입**으로 생성되는 **테스트의 품질**을 향상시킬 수 있습니다.

이러한 개선을 통해 AI TDD 에이전트가 더욱 신뢰성 높고 예측 가능한 파트너로 발전할 것을 기대합니다.
